## Patrones tecnicos aplicados

- **Pattern AAA (Arrange-Act-Assert)** en cada test, separado visualmente por bloques.
- **JUnit 5 (`@Test`, `@DisplayName`, `@BeforeEach`, `@ExtendWith(MockitoExtension.class)`)** para estructura semantica y reporte legible.
- **Mockito 5 (`@Mock`, `@InjectMocks`, `when().thenReturn()`, `verify()`)** para aislar los servicios de sus colaboradores (Repositories, Mappers, JwtTokenProvider).
- **AssertJ (`assertThat(...)`, `assertThatThrownBy(...).isInstanceOf(...).hasMessageContaining(...)`)** para aserciones fluidas y mensajes de error claros.
- **DisplayName en espanol con prefijo `[exito]`, `[alternativo]`, `[error]`** para que el reporte muestre explicitamente la cobertura de los 3 escenarios obligatorios.

## Reproducibilidad

Las pruebas se ejecutan de manera identica en cualquier maquina con Maven + Java 21 o solo Docker:

```bash
# Opcion A · Maven instalado
cd tb2/backend/application
./mvnw test

# Opcion B · Maven en contenedor (cero setup)
cd tb2/backend/application
docker run --rm -v "$(pwd):/build" -w /build maven:3.9-eclipse-temurin-21 mvn -DskipITs test
```
