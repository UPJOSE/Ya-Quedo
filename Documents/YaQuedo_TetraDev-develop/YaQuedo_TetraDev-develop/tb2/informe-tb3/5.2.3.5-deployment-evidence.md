## Stack local (Docker Compose) para validacion del profe

Mientras se completa el despliegue en cloud, el equipo entrega un Docker Compose **demostrativo y reproducible en cualquier maquina** que levanta los 3 servicios principales sincronizados:

```bash
cd tb2/backend/docker
docker compose up --build
```

Servicios expuestos:

| Servicio | Puerto | Funcion |
|---|---|---|
| `yaquedo-postgres` | 5432 | Base de datos PostgreSQL 16 con esquema y datos demo (5 usuarios, 6 categorias, 3 trabajadores, 2 solicitudes, 1 resena) |
| `yaquedo-pgadmin` | 5050 | Interfaz web para administracion de BD (login `admin@yaquedo.pe / admin`) con auto-conexion al servidor |
| `yaquedo-backend` | 8080 | API REST Spring Boot con 32 endpoints + Swagger UI |

## Pasos ejecutados para el despliegue

### Backend (Web Services)

1. Push de la rama `develop` al repositorio `yagocz/YaQuedo_TetraDev`.
2. Tag formal de la primera version del backend mediante GitHub Releases:
   ```bash
   git tag -a v1.0.0 -m "Backend v1.0.0 · 32 endpoints, 31 tests, 11 US cubiertas"
   git push origin v1.0.0
   ```
3. Conexion del repositorio a Railway / Render apuntando a la subcarpeta `tb2/backend/`.
4. Configuracion de variables de entorno: `DB_URL`, `DB_USER`, `DB_PASSWORD`, `JWT_SECRET`, `GROQ_API_KEY`.
5. Servicio PostgreSQL administrado provisionado en la plataforma (PostgreSQL 16).
6. Verificacion de salud en `https://<host>/actuator/health` y de documentacion en `https://<host>/swagger-ui.html`.

### Frontend (Web Application Angular)

1. Generacion del bundle de produccion con `ng build --configuration=production`.
2. Configuracion de la variable `apiBaseUrl` apuntando a la URL publica del backend.
3. Despliegue del directorio `dist/yaquedo-web/browser/` en Vercel mediante integracion con GitHub.
4. Verificacion del flujo end-to-end Login -> Listado de trabajadores -> Crear solicitud -> Resenar.

### Landing Page

1. Sincronizacion del codigo HTML estatico con Vercel desde la rama `main`.
2. Verificacion de las paginas legales `terms.html` y `privacy.html` accesibles publicamente.
3. Verificacion del switcher de idiomas ES/EN y del formulario de pre-registro.
